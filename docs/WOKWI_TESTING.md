# Wokwi Child Testing

Automated tests for the ESP32 child firmware running inside a [Wokwi](https://wokwi.com) simulation. Tests cover HTTP routes, EEPROM config persistence, and the full UDP command set (PING, ACTION, LOAD_STEP, RUNNER_GO, STATUS_REQ).

---

## Prerequisites

### 1. Wokwi CLI

```powershell
npm install -g @wokwi/cli
```

Or download the binary from https://github.com/wokwi/wokwi-cli/releases.

### 2. Wokwi private gateway (wokwigw)

Required for host ↔ simulator network connectivity (HTTP and UDP forwarding).

Download `wokwigw` from https://github.com/wokwi/wokwigw/releases.

### 3. Wokwi token

A Wokwi account token is required for headless CI use. Set it as an environment variable:

```powershell
$env:WOKWI_CLI_TOKEN = "your-token-here"
```

Free accounts include 50 simulation-minutes/month. For unlimited CI use, upgrade to a paid plan.

## One-time setup

### Build ESP32 firmware for Wokwi

Wokwi requires `SECRET_SSID = "Wokwi-GUEST"` and `SECRET_PASS = ""`:

```powershell
# 1. Switch arduino_secrets.h to Wokwi credentials:
Copy-Item main\arduino_secrets.wokwi.h.example main\arduino_secrets.h

# 2. Compile for ESP32 (no upload, no version bump):
powershell.exe -ExecutionPolicy Bypass -File build.ps1 -Board esp32 -NoUpload
```

The compiled firmware lands at `build\esp32\main.ino.elf` — the path referenced in `wokwi/wokwi.toml`.

> **Note:** Restore your real `arduino_secrets.h` afterwards. It is gitignored and will not be overwritten by git.

## Running the simulation

### Step 1 — Start the private gateway

In a separate terminal:

```powershell
wokwigw
```

This starts the local bridge that forwards TCP/UDP between localhost and the simulated ESP32.

### Step 2 — Launch the Wokwi simulation

```powershell
wokwi-cli --timeout 120 --serial-log-file wokwi-serial.log wokwi/
```

The simulator boots the ESP32, connects to the virtual WiFi AP (`Wokwi-GUEST`), syncs NTP, and starts serving HTTP on port 80 / UDP on port 4210. The gateway forwards:

| Host | Sim |
|------|-----|
| `localhost:18080` | ESP32 port 80 (HTTP) |
| `localhost:14210` | ESP32 port 4210 (UDP) |

### Step 3 — Run the tests

In a third terminal, once the simulation is up (watch for `NTP synced` in the serial log):

```powershell
python tests/test_child.py
```

Or with explicit host/port arguments:

```powershell
python tests/test_child.py 127.0.0.1 18080 14210
```

## Running against real hardware

Point the test at the child's actual IP and standard ports:

```powershell
python tests/test_child.py 192.168.10.x 80 4210
```

## Test coverage

| Section | What is tested |
|---------|---------------|
| HTTP basic | `GET /`, `/status`, `/config`, `/favicon.ico` — status codes, Content-Length, JSON fields, HTML form fields |
| HTTP POST /config | Form submission, 303 redirect, persistence verified by re-reading `/config` |
| UDP PING/PONG | CMD_PING → CMD_PONG; magic, version, cmd, hostname format, payload size |
| UDP STATUS_REQ | CMD_STATUS_REQ → CMD_STATUS_RESP; payload size, field ranges |
| UDP ACTION | CMD_ACTION (solid) → STATUS_REQ verification; CMD_ACTION_STOP clears state |
| UDP action variants | Flash, Wipe, Off accepted and reflected in STATUS_RESP |
| UDP runner | LOAD_STEP × 2 → ACK with correct step index; RUNNER_GO starts runner; RUNNER_STOP stops it |
| UDP invalid packets | Wrong magic and truncated headers are silently ignored |

```

  diagram.json          — Virtual circuit: ESP32-DevKit-V1 + 8-LED NeoPixel strip (GPIO 2)

  arduino_secrets.wokwi.h.example  — Wokwi WiFi credentials (SSID="Wokwi-GUEST")

  esp32/
    main.ino.elf        — Compiled output (generated by build.ps1 -Board esp32)

  test_child.py         — Child HTTP + UDP test suite
```

## Troubleshooting

**Simulation never reaches `NTP synced`**
- Check that `wokwigw` is running. Wokwi-GUEST needs the gateway to reach the internet for NTP.

**Tests fail with "no response" on UDP**
- Confirm the gateway is running and forwarding port 14210 → 4210.
- Check `wokwi-serial.log` for crash output.

**`LOAD_STEP` ACK not received**
- The child only sends ACK if the `UdpHeader.magic` and `version` match. Confirm the test is using little-endian byte order.

**POST /config returns 0 (connection error)**
- The sim may have crashed. Check the serial log and restart the simulation.
